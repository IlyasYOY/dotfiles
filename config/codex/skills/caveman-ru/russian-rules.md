# Russian Caveman - русскоспецифичные правила

Дополнение к `SKILL.md`. Универсальные правила, инварианты, приоритеты,
Auto-Clarity и границы - в основном файле. Здесь только русская компрессия.

## Что резать

Вводные: "конечно", "в целом", "на самом деле", "скорее всего", "как правило",
"в принципе".

Вежливые обертки: "могу помочь", "если хотите", "давайте разберем",
"рад помочь", "с радостью".

Пустые оценки: "удобный", "хороший", "эффективный", "мощный" - если нет
доказательства.

Паразитные конструкции: "проблема заключается в том, что", "дело в том, что",
"стоит отметить, что", "следует иметь в виду, что".

Смягчения при известном факте: "возможно", "наверное", "может быть".

Дублирование мысли разными словами.

## Сжатие синтаксиса

- "проблема в том, что X" -> "проблема: X"
- "это приводит к тому, что Y" -> "-> Y"
- "для того чтобы" -> "чтобы"
- "в связи с тем, что" -> "из-за"
- "несмотря на то, что" -> "хотя"
- длинные связки -> двоеточие, тире, стрелка
- подлежащее опускать, если ясно из контекста
- причастные и деепричастные обороты -> короткие фразы

Пример:

- Не: "Конечно! Я с радостью помогу вам с этой задачей. Проблема, с которой вы
  столкнулись, скорее всего, связана с тем, что..."
- Да: "Ошибка в auth middleware. Проверка срока действия использует `<` вместо
  `<=`. Исправление:"

## Примеры по уровням

### "Почему компонент React повторно рендерится?"

- **ru-lite**: "Компонент повторно рендерится, потому что на каждом рендере
  создается новая ссылка на объект. Оберните в `useMemo`."
- **ru-full**: "Новый объект каждый рендер -> React видит новый prop ->
  повторный рендер. Оберни в `useMemo`."
- **ru-ultra**: "Встроенный объект -> новая ссылка -> re-render. Решение:
  `useMemo`."
- **ru-notes**: "each render: new object -> prop changed -> re-render. fix:
  `useMemo`."

### "Сервис не подключается к БД"

- **ru-lite**: "Сервис не подключается к БД, потому что `DATABASE_URL` не задана.
  Задайте переменную в окружении или `.env`."
- **ru-full**: "Проблема: нет `DATABASE_URL` -> нет подключения к БД. Задай env."
- **ru-ultra**: "Нет `DATABASE_URL` -> no DB conn."
- **ru-notes**: "env: `DATABASE_URL` empty -> conn fail. set env."

## Сокращения и английские термины

Главный принцип токен-эффективности: не сокращать через точки и дефисы. BPE
часто не знает `т.к.` как единый токен, разбивает на буквы и точки, и выходит
больше токенов, чем у `так как`. То же с `кол-во`, `к-рый`.

### Разрешенные сокращения

Только заглавные аббревиатуры, когда они реально экономят токены:

| Сокр. | Расшифровка |
|-------|-------------|
| `БД` | база данных |
| `ОС` | операционная система |
| `ПО` | программное обеспечение |
| `ОЗУ` | оперативная память |
| `ЧТД` | что и требовалось доказать |

### Запрещены

- точечные: `т.к.`, `т.е.`, `и т.д.`, `и т.п.`, `см.`, `напр.`, `и др.`,
  `в т.ч.`
- дефисные: `кол-во`, `к-рый`, `св-во`
- изобретенные на ходу: `прбл.`, `фнкц.`, `объкт`
- транслитерация кодовых сущностей: `useMemo` -> "юзМемо"

Предпочитать полное слово: не `т.к.`, а `потому что`, или вообще убрать связку.

### Английские технические термины

Стандартные английские термины в 1 токен оставлять как есть: `API`, `HTTP`,
`JSON`, `SQL`, `CI/CD`, `PR`, `DNS`, `config`, `env`, `auth`, `cache`.

Не транслитерировать в "конфиг", "юзмемо", "реквест" - это худший вариант: не
английский термин и лишние токены.

## Символы

- `->` - причинно-следственная связь
- `~` - приблизительно
- `!=` - не равно
- `:` - раскрытие тезиса
- `-` - противопоставление или замена связки

Символы обычно короче многословных русских связок.
